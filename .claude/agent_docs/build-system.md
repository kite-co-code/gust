# Build System

## Stack
- **Vite 7** with `laravel-vite-plugin`
- **Tailwind CSS v4** via PostCSS (`@import 'tailwindcss'`)
- **Biome** for JS/CSS linting
- Custom glob import plugins for CSS & JS (defined in `vite.config.js`)

## Entry Points

| Source | Output |
|--------|--------|
| `assets/main.js` | `public/build/main.js` |
| `assets/main.pcss` | `public/build/main-styles.css` |
| `assets/editor-scripts.js` | `public/build/editor-scripts.js` |
| `assets/editor-styles.pcss` | `public/build/editor-styles.css` |
| `assets/admin-scripts.js` | `public/build/admin-scripts.js` |

## Component Asset Bundling

Main entry points use glob imports to auto-include all component assets:

```js
// assets/main.js
import '../components/*/scripts.js'
```

```css
/* assets/main.pcss */
@import '../components/*/styles.pcss';
```

- `scripts.js` and `styles.pcss` in any component → bundled into main outputs
- Component files with other names (e.g., `slider.js`) → built as standalone entries

## Static Assets

`assets/static/` → copied to `public/` without processing.

## Asset Versioning

`public/build/manifest.json` maps source names to versioned output names.
WordPress enqueues via this manifest (handled by `laravel-vite-plugin` integration).

## CSS Architecture

`assets/styles/` organized as ITCSS layers:
- `1-theme/` — Design tokens, CSS custom properties
- `2-base/` — Normalize, base element styles
- `3-patterns/` — Reusable patterns (type styles, etc.)
- `4-utilities/` — Utility classes

## Color System

Generated by `dev-scripts/postcss-color-system.js` from `assets/theme-config.json`.

**CSS custom properties per color:**
- `--color-{name}` — hex value
- `--color-{name}--hsl` — HSL format
- `--color-{name}--foreground` — contrasting text color

**Utility classes:**
- `color-context-{name}` — sets background + foreground + focus + links
- `has-{name}-background-color` — WordPress block editor alias
- `foreground-from-{name}` — text color only

## Config Files

| File | Purpose |
|------|---------|
| `vite.config.js` | Entry points, plugins, HMR config |
| `postcss.config.js` | PostCSS plugin chain |
| `tailwind.config.js` | Tailwind v4 config |
| `biome.json` | JS/CSS linting rules |
| `pint.json` | PHP PSR-12 formatting |
| `config.json` | Framework runtime settings |
| `theme.json` | WordPress FSE/block editor settings |
